# Autoresearch Wrapper

**Status:** WORK-layer planning artifact. **Not** Record truth. **Not** approval.

## Trigger

Use this wrapper when a work-dev research question needs structured exploration and a visible checkpoint rhythm.

Do not use it to claim Grace-Mar has an autonomous research agent. Do not use it to update SELF, SKILLS, EVIDENCE, SELF-LIBRARY, Voice, or `recursion-gate.md`.

## Mode

Choose one:

- **optimization** - drive a measurable scalar or quality target, such as lower error, higher accuracy, faster runtime, clearer retrieval, or stronger evidence coverage.
- **discovery** - test a causal, mechanistic, strategic, or conceptual claim with measurements and source-backed observations.

## Inner Loop

Bounded WORK exploration:

- inspect sources,
- collect cited evidence,
- write the experiment protocol before the test,
- run small analyses only if needed and within the bounded pass,
- draft one research note,
- keep dated logs or run notes.

## Experiment Protocol

Before each test or analysis pass, write:

| Field | Value |
|-------|-------|
| **Question** | [what this pass tests] |
| **Prediction** | [expected outcome] |
| **Why this test** | [why it is the next useful wedge] |
| **Measurement** | [proxy metric, source comparison, or inspection criterion] |
| **Sanity check** | [baseline reproduction, data integrity, source quality, or scope check] |
| **Result label** | confirmatory / exploratory / null / inconclusive |

## Outer Loop Heartbeats

Pause for human checkpoint at these milestones:

1. **Question framed** - hypothesis, success metric, and stop conditions are clear.
2. **Sources gathered** - relevant sources are listed and obvious source gaps are named.
3. **First synthesis** - preliminary findings are written with uncertainty.
4. **Candidate decision** - decide whether the result stays WORK-only or deserves a gate candidate draft.

## Stuck Behavior

If the run stalls, do not keep thrashing. Choose one:

- return to source search,
- route back to ideation,
- narrow the proxy metric,
- reframe the hypothesis,
- halt as inconclusive with a dated note.

## Direction Decision

At the candidate-decision milestone, choose one:

- **deepen** - test a sub-hypothesis or mechanism,
- **broaden** - inspect an adjacent question,
- **pivot** - replace the premise because the baseline or source picture changed,
- **conclude** - stop because the current evidence is sufficient for the artifact,
- **halt** - stop because the run is inconclusive or not worth further work.

## Stop Conditions

Stop or narrow the run if:

- sources cannot be cited,
- the success metric is vague,
- the run cannot distinguish draft from implemented capability,
- the output starts implying Record truth,
- the operator cannot inspect the log.
