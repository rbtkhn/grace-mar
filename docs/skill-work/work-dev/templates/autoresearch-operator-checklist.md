# Autoresearch Operator Checklist

**Status:** WORK-layer checklist. **Not** Record truth. **Not** approval.

## Before The Run

- [ ] The question belongs in work-dev.
- [ ] The hypothesis or research question is written in one sentence.
- [ ] Bootstrap is complete: scope, initial sources, initial hypotheses, proxy metric, and baseline are named.
- [ ] Mode is chosen: optimization or discovery.
- [ ] The success metric is inspectable.
- [ ] The time budget or bounded pass is explicit.
- [ ] Source-of-truth files or URLs are listed.
- [ ] The expected output is a dated WORK research note.
- [ ] No `npx`, clone, symlink, package install, or upstream file copy is required for this run.
- [ ] The operator knows when the first heartbeat checkpoint occurs.

## Heartbeat Milestones

- [ ] Question framed.
- [ ] Sources gathered.
- [ ] First synthesis written.
- [ ] Candidate decision made.

## During The Run

- [ ] Each test or analysis pass has a written protocol: question, prediction, why this test, measurement, and sanity check.
- [ ] Results are labeled confirmatory, exploratory, null, or inconclusive.
- [ ] Stuck behavior is explicit: return to sources, route to ideation, narrow the metric, reframe, or halt.
- [ ] Direction decision is explicit: deepen, broaden, pivot, conclude, or halt.
- [ ] No continuous, overnight, or uncheckpointed autonomy is implied.

## Before Any Gate Candidate Draft

- [ ] The run has cited sources.
- [ ] The run has a dated WORK log or run note.
- [ ] The proposed target surface is named.
- [ ] Draft / documented-only / implemented status is explicit.
- [ ] Contradictions or uncertainty are preserved.
- [ ] The candidate does not imply Voice or Record learned autonomously.

## After The Run

- [ ] Leave inconclusive results WORK-only unless the human asks to stage them.
- [ ] Record the next wedge in the run note.
- [ ] If a full candidate is drafted, mark it as pending human review.
- [ ] Do not edit SELF, SKILLS, EVIDENCE, SELF-LIBRARY, Voice, `recursion-gate.md`, or `bot/prompt.py` from this checklist.
