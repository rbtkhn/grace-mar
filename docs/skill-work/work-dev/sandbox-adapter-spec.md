**Territory:** work-dev

**Date:** 2026-04-06
**Gap:** BUILD-AI-GAP-010
**Origin:** [positioning-governed-state-os.md](positioning-governed-state-os.md) layer 1 gap; external six-layer analysis.

## Problem

External sandbox runtimes (E2B, Daytona, Modal, local Docker, future substrates) let agents execute code in isolated environments. If Grace-Mar or OpenClaw delegates work to a sandbox, the governance model currently stops at the sandbox boundary: no receipts flow back, authority classes don't apply inside the sandbox, and observability is whatever the vendor provides.

The adapter does **not** build sandboxing infrastructure. It wraps external sandboxes with the governance model so that authority, receipts, and observability remain on our terrain.

---

## Design principles

1. **Governance over hypervisor.** We do not own compute; we own the rules about who may use compute and what trace they leave. The adapter is a policy wrapper, not a container runtime.

2. **Receipt-first.** Every sandbox invocation produces a structured receipt before the caller sees results. No silent execution.

3. **Authority delegation, not assumption.** The adapter checks who is requesting sandbox execution and whether that caller holds sufficient authority before forwarding to the runtime. The companion is always the ultimate gate.

4. **Substrate-agnostic.** The adapter defines an interface. Each sandbox backend implements it. Switching from E2B to Daytona or to local Docker changes the backend, not the governance layer.

5. **Companion-scale first.** The spec targets one-companion, one-operator use. Multi-tenant, multi-agent fleet orchestration is out of scope until the single-companion case is proven.

## Architecture

```
┌─────────────────────────────────────────────────┐
│  Caller (operator script, agent, integration)   │
└──────────────────┬──────────────────────────────┘
                   │  SandboxRequest
                   ▼
┌─────────────────────────────────────────────────┐
│              Sandbox Adapter                     │
│                                                  │
│  1. Authority check (who is calling, scope)      │
│  2. Pre-execution receipt (request logged)       │
│  3. Delegate to backend                          │
│  4. Capture result + artifacts                   │
│  5. Post-execution receipt (result logged)        │
│  6. Compute ledger row (cost, task_id)           │
│  7. Return SandboxResult to caller               │
└──────────┬──────────────────────────────────────┘
           │  Backend-specific API
           ▼
┌─────────────────────────────────────────────────┐
│  Sandbox Backend (E2B / Daytona / Docker / …)   │
└─────────────────────────────────────────────────┘
```

## Data shapes

### SandboxRequest

```yaml
request_id: ""         # UUID, generated by adapter
caller: ""             # who is invoking (e.g. "operator", "openclaw_hook", "agent:work-dev")
authority_class: ""    # operator | agent_supervised | agent_autonomous
task_id: ""            # links to compute-ledger task_id
task_type: ""          # e.g. "code_execution", "test_run", "build", "analysis"
command: ""            # shell command or script path
files_in: []           # list of { path, sha256 } — files sent into sandbox
timeout_ms: 0          # max execution time; 0 = backend default
backend: ""            # e2b | daytona | docker | local
backend_config: {}     # backend-specific overrides (image, resources, env)
record_access: ""      # none | read_only | read_write (what Record files the sandbox may see)
```

### SandboxResult

```yaml
request_id: ""         # matches SandboxRequest
backend: ""
exit_code: 0
stdout: ""             # truncated if over size limit
stderr: ""
files_out: []          # list of { path, sha256, size_bytes } — artifacts produced
wall_ms: 0
backend_cost: {}       # backend-reported cost (tokens, compute-seconds, credits — shape varies)
error: ""              # non-empty if execution failed outside sandbox (auth, timeout, backend down)
```

### SandboxReceipt

Written to `pipeline-events.jsonl` (or a dedicated `sandbox-receipts.jsonl` if volume warrants).

```yaml
event_type: "sandbox_execution"
request_id: ""
caller: ""
authority_class: ""
task_id: ""
task_type: ""
backend: ""
record_access: ""
exit_code: 0
wall_ms: 0
files_in_count: 0
files_out_count: 0
files_out_sha256: []   # artifact hashes for provenance
success: true
error: ""
ts: ""                 # ISO-8601
```

### Compute ledger row

Uses the existing `append_integration_ledger()` shape (GAP-009 extended):

```json
{
  "bucket": "sandbox",
  "operation": "sandbox_execution",
  "runtime": "e2b",
  "task_id": "TASK-042",
  "task_type": "code_execution",
  "success": true,
  "wall_ms": 3200,
  "bytes_processed": 14500,
  "outcome_confidence": 0.9
}
```

## Authority classes

| Class | Who | What it allows | Gate requirement |
|-------|-----|----------------|------------------|
| `operator` | Human operator in Cursor / CLI | Any sandbox invocation; full Record read | None (operator is the authority) |
| `agent_supervised` | Agent acting under operator watch (e.g. Cursor agent mode) | Sandbox invocation within declared scope; Record read_only | Operator must be in the loop; receipts reviewed post-hoc |
| `agent_autonomous` | Agent acting without real-time operator supervision | Restricted to pre-approved task_types; no Record write; timeout enforced | Pre-approval required; receipts auto-flagged for review |

The adapter **refuses** requests where `authority_class` does not match the declared scope. For example, `agent_autonomous` cannot request `record_access: read_write`.

## Backend interface

Each backend implements a minimal Python protocol:

```python
from typing import Protocol

class SandboxBackend(Protocol):
    name: str

    def execute(
        self,
        command: str,
        files_in: list[dict],
        timeout_ms: int,
        config: dict,
    ) -> dict:
        """Run command in sandbox; return { exit_code, stdout, stderr, files_out, wall_ms, cost }."""
        ...

    def health(self) -> dict:
        """Return { ok: bool, notes: str }."""
        ...
```

Initial backends (stubs only until needed):

- **`LocalDockerBackend`** — wraps `docker run` with volume mounts for files_in/out
- **`E2BBackend`** — wraps E2B API (requires `E2B_API_KEY`)
- **`DaytonaBackend`** — wraps Daytona API
- **`DryRunBackend`** — logs the request and returns a mock result (for testing governance without real execution)

## Governance integration points

| Existing primitive | How the adapter uses it |
|--------------------|------------------------|
| **Capability contract** (GAP-008) | Each backend gets a `capability-contract-sandbox-*.yaml` describing its auth, failure policy, cost, and governance posture |
| **Compute ledger** (GAP-009) | Every execution appends a row with `bucket: sandbox`, `task_id`, `task_type`, `outcome_confidence` |
| **Pipeline events** | Pre- and post-execution receipts written to `pipeline-events.jsonl` |
| **Harness events** | Optional `sandbox_execution` event for harness audit trail |
| **Constitution / INTENT** | If the sandbox produces artifacts that will be staged to RECURSION-GATE, the constitution advisory runs at staging time (existing `openclaw_stage` path), not at sandbox invocation |
| **Continuity contract** | Sandbox execution does not bypass continuity requirements; if results flow to handback, continuity receipts are still required |

## What this spec does NOT cover

- **Building a container runtime.** We use existing runtimes.
- **Agent-to-agent communication.** The adapter serves one caller per invocation; multi-agent orchestration is a separate concern (layer 6).
- **Billing/provisioning policy.** The compute ledger records cost; budget enforcement and approval thresholds are future work (adjacent to GAP-009 extension).
- **Persistent sandbox state.** Each invocation is stateless. If persistent environments are needed, that is a backend feature, not an adapter concern.
- **File content inspection.** The adapter hashes files for provenance but does not inspect or filter content. Content governance is the caller's responsibility.
