# Cheat sheet: Polyphonic cognition protocol

**Purpose:** One-page training for staff (or principal) on when and how to use the assistant brain. Principal owns the decision; the protocol supports clarity, not recommendation.

**Related:** [state-evolved-assistant-brain-heads-of-state.md](state-evolved-assistant-brain-heads-of-state.md), [prep-before-call-abc.md](prep-before-call-abc.md), [polyphonic-cognition-protocol-skill.md](polyphonic-cognition-protocol-skill.md).

---

## When to query

- **Before an important call** (client, federal, partner, investor).
- **Before a decision** that has legitimacy, structure, or liability dimensions.
- When you want the **full field** in one place — no extra meetings to gather every perspective.

**Time:** 5–10 minutes. One query; read A/B/C and tensions; decide.

---

## What you get

- **A — Legitimacy / orientation** (narrative, how it will be read).
- **B — Structure / constraints** (power, institutions, constraints).
- **C — Liability / mechanism** (who's on the hook, who defects first, reversibility).
- **Reversibility:** What becomes irreversible.
- **Tensions:** Where perspectives conflict (we label; we do not resolve).

**Optional:** Say your instinct first ("Where are you leaning?") — the system then shows where your read sits relative to A/B/C.

---

## Rules (for you)

1. **You decide.** The system never gives one answer. It surfaces options; you choose.
2. **Relay only if you say so.** Nothing is written to any record unless you explicitly say "save" or "record."
3. **After the call or decision:** If it's Record-worthy, say **"we did X"** (e.g. "We had a call with [who] about [what]") so it stages to the gate / session log. See [we-did-x-habit.md](../../we-did-x-habit.md).

## One-line reminder

**Decide:** You choose; the system doesn't.
