# Scenario: Using the polyphonic cognition protocol skill

A short illustration of how the skill is used in practice—before a high-stakes call or decision.

---

## Setup

**Jordan** is a principal’s chief of staff. Tomorrow there’s a 30-minute call with a key ally who has been lukewarm on a pending endorsement. The principal is weighing whether to ask directly for the endorsement on the call or to use the time only to “check in and listen.” Jordan has 20 minutes to prep. They open the assistant (OpenClaw, Cursor, or another agent) with the polyphonic cognition protocol skill loaded.

---

## 1. Invoking the skill

Jordan could ask in several ways; any of these trigger the protocol:

- *"Give me A/B/C on whether we should ask for the endorsement on tomorrow’s call or just do a listening check-in."*
- *"Pre-call prep: 30-min call with [ally]. We want their endorsement but they’ve been hesitant. Run the protocol."*
- *"I’m leaning toward asking directly—give me the angles and the civ-mem lens."* (instinct-as-input + D)

Jordan chooses the third. They state their leaning and ask for the full field including the civ-mem lens, so the run includes **D**.

---

## 2. What the agent returns (illustrative)

The agent responds with four labeled blocks (A, B, C, D), reversibility, and tension—no single recommendation. Because Jordan asked for the civ-mem angle, **D — Civ-mem lens** is included.

**Context:** 30-min call with key ally; principal weighing whether to ask for endorsement now vs. check-in only.

**A — Legitimacy / orientation**
- Asking directly shows confidence and clarity; the ally can say yes or no and the relationship stays defined.
- A “listening only” frame can be read as diffident or as if the principal doesn’t want to put them on the spot; it depends how the ally reads the relationship.
- If the principal never asks, the ally may never have to commit; the “ask” itself is a signal of seriousness.

**B — Structure / constraints**
- One 30-min slot; if the ask happens late, there’s little time to respond or negotiate.
- The ally has their own internal stakeholders; a public endorsement may require them to move others first.
- Calendar and follow-up: a soft “we’ll follow up” leaves the door open but defers the moment of decision.

**C — Liability / mechanism**
- If we ask and they say no, we’ve burned the ask for this cycle; we can’t “un-ask.”
- If we don’t ask, we keep the option open for a later, better moment—but “later” may never come.
- Reversibility: Asking and getting a no is hard to reverse with this ally; not asking is reversible (we can ask next time).

**D — Civ-mem lens**
- **One subject, many tongues:** The "ask" is one moment; the ally's tradition (institutional, political, personal) may name it differently—endorsement as commitment, as signal, as risk. Holding both readings in view.
- **Seam:** "We want their yes" vs "we don't want to burn the relationship." The seam isn't resolved; the protocol makes it visible so the principal can choose with both in mind.
- **Face vs category:** Are we addressing the ally as a face (relationship, trust) or as a category (endorser, bloc)? The call might sit at the seam between the two.
- **Abundance:** Is there a read where both sides gain (e.g. clarity for us, a clear ask for them) rather than a fixed pie? **Growth:** What do we learn from this call either way—yes, no, or defer—so the moment builds capability, not only success/failure.

**Reversibility:** Asking and receiving a no (or a vague demurral) is the main irreversible outcome; the relationship shifts. Not asking keeps the status quo reversible.

**Tensions:** A and C conflict here. A leans toward asking (clarity, definition); C leans toward preserving the option and avoiding a no. We do not resolve.

**Your lean sits closer to A (ask directly).** B would push back by: “Structure says one short slot—consider a clear ask early so there’s time to react, or name a specific follow-up.” C would push back by: “If they’re not ready, the no is costly; consider a softer ‘where do you see this going?’ and leave the explicit ask for a dedicated follow-up.”

---

## 3. How Jordan uses it

Jordan reads A, B, C, and D and the tension line. They don’t get “do this” or “don’t do this”—they get the full field. They see their instinct (ask) mapped to A, with B, C, and D offering different angles. They use it to:

- Brief the principal in two minutes: “Here are the four angles—legitimacy, structure, liability, and the civ-mem lens. The main tension is ask now for clarity vs. preserve the option and avoid a no. D reminds us: seam between wanting their yes and not burning the relationship; face vs category in how we address them.”
- Propose a middle path: “We could open with a direct but short ask in the first 10 minutes, so there’s time to hear them out—B’s point—without burning the relationship if they’re not ready—C’s point.”
- Leave the final call to the principal.

After the call, Jordan (or the principal) says **“We did X”** (e.g. “We asked in the first 10 minutes; they said they’d get back to us by Friday”). That can feed into whatever logging or pipeline the team uses; the skill itself does not write anything unless the user explicitly says “record” or “save.”

---

## 4. Why this illustrates the skill

| Element | In the scenario |
|--------|------------------|
| **No single answer** | The agent never says “you should ask” or “you shouldn’t.” It gives A, B, C, D and a tension. |
| **Fixed perspectives** | A = legitimacy/narrative, B = structure/constraints, C = liability/mechanism, each applied to *this* decision. |
| **Instinct-as-input** | Jordan stated a lean; the response mapped it to A and showed how B and C would push back. |
| **Reversibility** | One line makes explicit what becomes hard to undo (asking and getting a no). |
| **Tension visible** | “A and C conflict here. We do not resolve.” The user sees the tradeoff; the system doesn’t collapse it. |
| **Sovereign session** | Nothing is saved unless the user says “record” or “save”; “we did X” is a separate habit for their own process. |

---

## How civ-mem is tied in

The protocol **comes from** civ-mem: it is the **STATE** operating mode (present-moment options, tensions preserved, no write to the long-term record unless the user explicitly relays) turned into a loadable skill. A/B/C map to the civ-mem **three minds** (legitimacy, structure, liability). **D** is the civ-mem lens: when the user has the civ-mem corpus or asks for the civ-mem/condition/seam angle, the agent adds D (condition, seam, one subject many tongues, face vs category) applied to the decision. Polyphonic cognition—holding multiple perspectives without collapsing to one answer—is how STATE behaves; the scenario above shows one integrated run with A/B/C/D.
