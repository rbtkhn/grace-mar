# work-dev mirror — use now vs later

## Use now (Week 1-2)

- **Gate discipline first:** Stage candidates in `recursion-gate.md`; never hand-merge identity files.
- **Continuity habit:** Before major tasks, read recent `session-log.md` and current `recursion-gate.md`.
- **Provenance habit:** For claims in drafts, keep source file names in notes.
- **Boundary habit:** Keep WORK documents in WORK paths; Record truth only through approval flow.

## Use soon (Week 3-4)

- **Structured handback style:** Ask the agent to output what changed, why, evidence source, and pending risks.
- **Reliability pass:** Run a lightweight pre-ship checklist on high-stakes outputs.
- **Escalation discipline:** When uncertain, pause and request operator review instead of shipping.

## Use later (Month 2+)

- **Terminal-backed checks:** continuity scripts, integrity checks, and CI-facing routines.
- **Scenario stress testing:** factorial variation and targeted failure-mode checks.
- **Advanced integration:** OpenClaw handback conventions and deeper observability patterns.

## Mapping to Xavier learning objectives

- **LO-02 / LO-06:** Gate + boundary habits
- **LO-03 / LO-04:** Provenance + stress-test habits
- **LO-05:** Continuity and weekly operating rhythm
