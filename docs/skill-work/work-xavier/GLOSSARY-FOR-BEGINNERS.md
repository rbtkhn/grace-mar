# Glossary for beginners — Xavier

Short, plain-language definitions used in `work-xavier`.

---

- **Record**  
  The durable identity files for a companion (what is treated as true after approval).

- **Gate / recursion-gate**  
  The review queue where proposed identity updates are staged before approval.

- **Candidate**  
  A proposed new item waiting for approve/reject decision.

- **Stage**  
  Put proposed updates into the gate queue; not yet merged into Record.

- **Merge (gated)**  
  Apply approved candidates into Record files using the merge script flow.

- **WORK docs**  
  Planning/execution docs for projects. Useful, but not identity truth by default.

- **Stress-test**  
  Quick risk check for high-stakes drafts: PASS/HOLD/FAIL.

- **PASS / HOLD / FAIL**  
  PASS = can ship after human review.  
  HOLD = pause and escalate.  
  FAIL = do not ship.

- **Source integrity**  
  Every factual claim has a real citation.

- **Unofficial voice**  
  Independent analysis tone; not speaking as campaign office.

- **Mirror**  
  Shared docs replicated across workspaces to collaborate. Does not mean copying Record truth between forks.

- **Leakage**  
  Accidental copy of one companion’s identity content into another companion’s files.

- **Lane**  
  Path ownership boundary for PR/work scope (keeps unrelated changes separated).
