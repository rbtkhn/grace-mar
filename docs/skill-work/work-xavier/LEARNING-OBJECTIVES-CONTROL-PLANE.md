# Learning Objectives Control Plane — Xavier

**Purpose:** Bring `work-dev` rigor into `work-xavier` in beginner-safe form.  
**Use:** Weekly review of learning progress, behavior quality, and safe execution.

---

## 1) Learning objectives (what “good” looks like)

| Objective ID | Objective | Observable behavior | Evidence artifact |
|---|---|---|---|
| LO-01 | Navigate workspace confidently | Opens correct docs in order without operator rescue | Session notes + checklist completion |
| LO-02 | Follow gate discipline | Stages in `recursion-gate.md`; does not hand-edit `self.md` for new claims | Gate diff + approval log |
| LO-03 | Source integrity in public drafts | Every factual claim has receipt link before ship | Post cards + source check table |
| LO-04 | Stress-test judgment | Uses PASS/HOLD/FAIL correctly on high-stakes drafts | Stress-test register entries |
| LO-05 | Weekly operating rhythm | Completes plan -> ship -> KPI -> retro cycle | Week execution + KPI/budget log |
| LO-06 | Boundary literacy | Distinguishes WORK docs vs Record truth vs template docs | Verbal check + no leakage incidents |

---

## 2) Behavior scorecard (Green / Yellow / Red)

| Area | Green | Yellow | Red |
|---|---|---|---|
| Gate discipline | No hand merges; approvals before apply | Minor confusion, corrected before merge | Identity edits bypass gate |
| Source integrity | 100% sourced factual claims | 1-2 partial citations fixed pre-ship | Unsourced factual claim shipped |
| Stress-test use | High-stakes posts have PASS/HOLD/FAIL | One missing test, no harmful ship | High-stakes ship without test |
| Budget discipline | Weekly spend <= cap with notes | Approaches cap without forecast | Exceeds cap without escalation |
| Process ownership | Xavier runs daily/weekly flow | Needs frequent nudges | Flow stalls or is skipped |
| Boundary safety | Zero leakage incidents | Near miss, caught in review | Confirmed cross-fork leakage |

## 3a) Daily bridge from good morning to objectives

Each daily `good morning` should include one suggested `work-xavier` development path:
- map it to exactly one LO (`LO-01`..`LO-06`)
- map it to exactly one business-skill outcome
- define 1-2 same-day observable completion checks

This keeps day-level focus aligned with week-level outcome tracking.

For Week 1 startup, choose the initial LO using Session 0 seed survey themes (especially Section F business dream/objective/resources and Section E/G behavior/risk preferences).

---

## 4) Intervention rules (if stuck)

- **Two Yellow weeks in same objective:** move to focused micro-drill in next week plan.
- **Any Red in gate discipline or boundary safety:** immediate operator review before next ship.
- **Any Red in source integrity:** freeze public posting until source process is corrected.
