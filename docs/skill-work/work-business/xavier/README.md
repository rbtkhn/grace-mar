# work-business/xavier

**Purpose:** Optional **WORK territory** folder for business documents and execution planning **for an operator workflow** in the grace-mar repo.

**Instance Record:** A companion who bootstraps from [companion-self](https://github.com/rbtkhn/companion-self) maintains **their own repository**; identity truth and `recursion-gate.md` live **there**, not under `users/` in grace-mar.

---

## What goes here

- Raw business docs (briefs, offers, SOPs, calendars, KPI sheets) that the operator stages for WORK-only use in this repo
- Short source index with file names and what each file is for
- Extracted objectives, constraints, and next actions for weekly execution

---

## Minimum starter pack

Create or update these files first:

1. `source-index.md` — what docs exist and where they came from
2. `objectives-and-constraints.md` — 90-day objective, bottlenecks, available resources
3. `week-0-priority-stack.md` — top 3 business priorities for the next 7 days

---

## Guardrails

- Do not copy from `users/grace-mar/**`.
- Do not treat WORK notes as Record truth.
- If a WORK insight should affect **someone’s** identity, that person’s instance stages a candidate in **`their` repo’s** `recursion-gate.md` and waits for approval — not in grace-mar unless it is the grace-mar companion’s Record.
