# CIV-MEM Topic Trace Contract

**Classification:** **WORK · DERIVED · NOT RECORD**

This contract governs CIV-MEM topic tracing inside Grace-Mar.

---

## Purpose

CIV-MEM topic tracing is a Grace-Mar WORK-layer method for deriving receipt-backed historical, institutional, legal, literary, geopolitical, and civilizational analogues from an external corpus. It is for operator reasoning, not automatic identity truth.

---

## Read-only upstream boundary

- Grace-Mar may read pinned upstream material in `research/repos/civilization_memory/`.
- `civilization_memory` remains external and read-only from this lane's contract perspective.
- This lane does not edit, patch, or require commits to upstream `civilization_memory`.
- All integration logic, routing contracts, templates, dashboards, and derived trace artifacts live in Grace-Mar.

---

## Structural analogy is not truth validation

- Structural analogy is a reasoning aid, not doctrine.
- CIV-MEM traces do not validate metaphysical, theological, ideological, or speculative truth claims.
- Pattern resemblance, historical rhyme, or institutional similarity must never be presented as proof of external metaphysics.

---

## Allowed trace behaviors

- Read pinned CIV-MEM upstream reference material.
- Route an operator topic to seed files.
- Traverse MEM CONNECTIONS to expand the trace graph.
- Summarize graph spine and major branch logic.
- Identify analogues, gaps, and stress tests.
- Produce Grace-Mar-local derived WORK artifacts (notes, receipts, route logs, trace summaries).
- Propose optional Grace-Mar gate candidates for companion review when a durable claim is intentionally promoted.

---

## Prohibited trace behaviors

- Editing `civilization_memory` files from this lane contract flow.
- Treating CIV-MEM as doctrine or canonical truth authority for Grace-Mar.
- Claiming CIV-MEM validates external metaphysics or theology.
- Merging trace conclusions directly into SELF or Record surfaces.
- Silently promoting WORK artifacts into canonical state.

---

## Promotion boundary

- Default destination is WORK-only artifacts.
- Any promotion beyond WORK requires explicit human decision through normal Grace-Mar gate flow.
- Proposed candidates must be explicit, reviewable, and separately approved.
- No trace output is canonical by default.

---

## Minimal example

**Operator topic:** "Trace this theological idea through history."  
**Trace flow:** route topic to seed materials -> traverse connections -> summarize analogues/gaps/stress tests in a Grace-Mar WORK note.  
**Optional promotion:** if a durable companion-facing claim appears, stage an optional gate candidate; do not merge directly into SELF/Record.
