# Moonshot — lifecycle

**Status:** WORK only. [Source policy](moonshot-source-policy.md) · [boundaries](moonshot-boundary.md).

| # | Stage | Input | Output | Allowed action | Forbidden action |
|---|--------|--------|--------|----------------|------------------|
| 1 | **Discovery** | Public signal, tip, or operator note; optional links. | One-line **label** + link bundle (WORK). | **Log** a signal; mark **tentative** tier. | **Assert** as fact; **merge** to Record. |
| 2 | **Source classification** | Raw links, titles, and notes. | **Tier 1–4** (source) + **A–D** (claim) per [source policy](moonshot-source-policy.md). | **Classify** downward if ambiguous. | **Promote** Tier 3/4 to “proven” without Tier 1/2. |
| 3 | **Contribution mapping** | Classified signal + [map template](moonshot-map-template.md). | Filled **Contribution Map** (WORK file). | **Edit** only WORK paths in-repo. | **Imply** SELF/EVIDENCE change without a **candidate**. |
| 4 | **Boundary check** | Filled map. | **Y/N** answers to **§6** in template + notes. | **Block** or **flag** on any YES to Record touch w/o gate. | **Override** [moonshot-boundary](moonshot-boundary.md) for convenience. |
| 5 | **Experiment framing** | Map **§7**; evidence posture. | **Smallest** experiment with **success/fail** signals. | **Design** a **reversible** step. | **Launch** off-repo spend or contracts **unapproved**. |
| 6 | **Proposal staging** | Framed experiment + need for durable change. | `CANDIDATE-*` or operator gate block **text**; evidence bundle list. | **Draft**; **link** in agenda. | **Auto-merge**; **bypass** companion review. |
| 7 | **Human decision** | Candidate in inbox / gate. | **Approve** / **defer** / **reject** (recorded in gate flow). | **Decide** as **human** or **companion** per repo rules. | **Model**-only “approval.” |
| 8 | **Execution** (only if separately authorized) | Approved plan + clear scope. | **Receipts** (WORK), optional EVIDENCE **after** approved merge. | **Execute** only what was **authorized**. | **Scope creep** into Record w/o new candidate. |
| 9 | **Review and receipt** | Outcome vs. success signals. | Short **receipt** in WORK; update [dashboard](moonshot-dashboard.md). | **Close** the loop; **downgrade** claims if failed. | **Silently** keep hype-tier claims. |
| 10 | **Optional promotion candidate** | Stable judgment worth surfacing. | New **CANDIDATE-*** for the right **surface** (e.g. STRATEGY, SELF-LIBRARY chapter). | **Propose** with **provenance** chain. | **Direct** file edit of canonical **SKILLS** or **EVIDENCE** from here. |
