# Multi-frame review (procedure)

**Status:** WORK only.  
**Scope:** **How** to run a multi-perspective pass — not a registry of frames.

---

## Rule

Important questions should be viewed through **at least two to three** distinct frames **before** final synthesis. Final synthesis should preserve **productive tensions** when they carry information.

**Frame definitions** live in a **per-lane** manifest or table. Do not duplicate lens content in this file.

---

## Procedure

1. **Shared input** — Same neutral summary or artifact for every frame (no lens-specific cherry-picking before frames run).  
2. **Per frame** — Short output: what this frame emphasizes, what it minimizes, what it would test next.  
3. **Cross-cut** — Where do frames agree? Where do they disagree? Is disagreement informative?  
4. **Synthesis** — Lane judgment: what matters, what is uncertain, what to do next — explicitly carry tension forward when useful.  

---

## When to run

- Before high-stakes synthesis or public ship.  
- When a single frame feels “too easy” or too aligned with prior bias.  
- After a **framing audit** when the audit says **core model** or **high** usefulness — still run multiple frames if stakes warrant.
