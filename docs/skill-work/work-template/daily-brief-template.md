# Daily brief — [LANE] (generic semantic template)

**Purpose:** One dated (or rolling) **operator surface**: snapshot + focus + fast/slow + optional emerging pattern + inputs + themes + multi-frame pass + synthesis + next actions.

**Boundary:** WORK only. Not Record truth. Human sign-off before client-facing or public ship.

**Important:** This file uses **semantic section titles only** — no fixed `1b` / `1d` numbering. Each lane assigns numbers (or dates) locally. **work-strategy** uses a numbered combined brief; see mapping below.

**SSOT:** The authoritative numbered spec for the shared work-politics + work-strategy daily brief is [work-strategy/daily-brief-template.md](../work-strategy/daily-brief-template.md). This generic scaffold is for cross-lane pattern reuse only.

---

## Lane snapshot

_Compact view of the lane “today”: status, blockers, what changed._

## Lane focus

_What this lane is prioritizing (bullets or short prose)._

## Fast vs slow horizon

- **Fast:** Inputs that turn on hours–days (e.g. headlines, tickets, chat).  
- **Slow:** Structural context that turns on weeks–months (e.g. research layer, roadmap, district fundamentals).  

## Emerging pattern

_Optional. One credible early pattern, or explicit “none this cycle.” Use [emerging-pattern-template.md](emerging-pattern-template.md). If the pattern uses a strong analogy, run [framing-audit-template.md](framing-audit-template.md)._

## Inputs / signals

_RSS, logs, events, links — lane-specific._

## Lead themes

_What threads matter from the inputs (often split by sub-lens, e.g. campaign vs product)._

## Multi-frame review

_Run per [multi-frame-review.md](multi-frame-review.md). Summarize convergence and productive tension._

## Synthesis

_Lane-level judgment: what matters, uncertainty, risks._

## Next actions

_Concrete operator steps; link to gate candidates if staging._
