# cici-ai First Task Proof Packet

**Status:** WORK / measurement template  
**Scope:** cici-ai community activation  
**Boundary:** This file defines operator evidence standards only. It does not alter Cici's Record.

---

## 1. Purpose

Define what counts as "first task completed" so activation claims are evidence-linked and repeatable, not narrative impressions.

## 2. Valid proof types

- GitHub issue opened
- GitHub PR opened
- repo cloned + setup note
- prompt pack created
- notebook or markdown artifact created
- guided task completion note
- screenshot or operator evidence pointer, if no public artifact exists
- peer/helper confirmation

## 3. Minimum proof packet

- participant alias or non-sensitive identifier
- week of
- task type
- artifact link or evidence pointer
- evidence confidence: A / B / C
- owner/reviewer
- follow-up needed
- graduation status: not yet / candidate helper / helper

## 4. Confidence tags

- **A** = direct artifact or repo-visible evidence
- **B** = direct operator observation or private screenshot/evidence pointer
- **C** = indirect claim, self-report, or inferred activity requiring confirmation

## 5. Not valid by itself

- joined group
- reacted to post
- vague enthusiasm
- private claim with no evidence pointer
- one-time message without task output
