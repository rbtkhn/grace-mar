# cici-ai Lane Roster Format

**Status:** WORK / roster template  
**Scope:** `cici-ai` team lanes and contribution tracking  
**Boundary:** This is a coordination template only. It does not replace the dashboard or proof packet.

---

## Purpose

Use this format to track how each member contributes across the `cici-ai` work project lanes:

- Design
- Application
- Core

Members may contribute to more than one lane. The roster should make that easy to see without locking anyone into a single identity.

---

## Roster table

```md
| Name | GitHub | Country | Primary lane | Secondary lane | Current status | Evidence level | Notes |
|---|---|---|---|---|---|---|---|
|  |  |  |  |  |  |  |  |
```

---

## Field guidance

- **Name**: display name or chosen team name
- **GitHub**: public handle or repo link
- **Country**: self-reported or operator-confirmed country
- **Primary lane**: Design, Application, or Core
- **Secondary lane**: optional second lane if the member contributes there too
- **Current status**: invited, joined, introduced, forked, goal stated, implementing, active, blocked, helper
- **Evidence level**: A / B / C
- **Notes**: short context, open loops, or next prompt

---

## Suggested status flow

invited  
-> joined  
-> introduced  
-> forked  
-> goal stated  
-> implementing  
-> active  
-> helper

---

## Lane hints

- **Design**: clarity, onboarding, presentation, examples, docs polish
- **Application**: personal goals, repo implementation, practical use cases
- **Core**: repo conventions, governance, reporting, templates, workflow stability

---

## Use rules

- Keep the table short enough to maintain weekly.
- Use evidence levels when a status is not fully confirmed.
- Allow people to move between lanes as they grow.
- Treat the roster as a contribution map, not a permanent label.
