# cici-ai Reviews

**Status:** WORK / review instances  
**Scope:** weekly cici-ai community activation and governance review  
**Boundary:** These reviews are Grace-Mar advisor-lane records. They do not update Cici’s Record.

## Purpose

- Store dated weekly review instances.
- Link activation metrics to evidence.
- Surface blockers, drift flags, and candidate actions.
- Preserve the distinction between work-cici advisory notes and cici-ai governed-state changes.

## Naming

- `cici-ai-weekly-review-YYYY-MM-DD.md`
- Date should be the week-of date.

## Rules

- Do not invent counts.
- Every nonzero count should have evidence pointer support.
- Use A/B/C confidence tags:
  - **A** = direct artifact or repo-visible evidence
  - **B** = direct operator observation or private evidence pointer
  - **C** = indirect claim, self-report, inferred activity, or aggregate evidence requiring confirmation
- Any durable identity, policy, or governed-state change for Cici must become a candidate proposal in cici-ai, not a direct work-cici edit.
