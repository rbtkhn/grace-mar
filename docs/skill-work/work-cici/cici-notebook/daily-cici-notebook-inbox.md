# Daily Cici notebook inbox (accumulator)

**Purpose:** **Append-only** scratch surface for the **current local calendar day** while you track OB1/Cici, coaching notes, or links — **before** they land in the per-day [`inbox/`](inbox/README.md) file the digest ingests.

**Fold at `dream` (or operator manual direction):** Synthesize rolling content into **`inbox/YYYY-MM-DD.md`** for that date (merge with anything already there; do not duplicate raw dump). Then run or schedule [`cici_journal_ob1_digest.py`](../../../../scripts/cici_journal_ob1_digest.py) as you already do for the day file. **Assistants:** keep captures **here** until **`dream`** or the operator **explicitly** says to fold into `inbox/` — do not merge the rolling file into `inbox/YYYY-MM-DD.md` on casual capture alone. The rolling file is **not** automatically cleared each dream; clear or trim manually when you want a fresh buffer.

**Length (scratch section only — below the append line):** When the scratch body exceeds **~20000 characters**, **prune from the top** (oldest lines first) in **~5000-character blocks** until **≤ ~20000 characters** remain (repeat if a single paste still leaves you above the limit). Re-count after large pastes.

**Contrast:** [`inbox/YYYY-MM-DD.md`](inbox/README.md) is the **digest input** for that calendar day. This file is a **volatile buffer** — same pattern as [strategy-notebook daily-strategy-inbox](../../work-strategy/strategy-notebook/daily-strategy-inbox.md).

**Git:** Commits preserve history.
