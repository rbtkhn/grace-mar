## 0) Necessary content

**Two surfaces:** a **pinned** post (durable group contract) and a **first @all** line that no one can miss. They overlap on purpose; the pin leans **rules and safety**, the broadcast leans **welcome and momentum**.

| Must appear (why it matters) | In pin (§1) | In first @all (§2) |
|---|---|---|
| **Open Brain (OB1) named and defined in one breath** | Yes | Yes |
| **Purpose of this group** - who it is for, what problem it solves | Short | Expanded |
| **Cici public instance as the golden path** | Yes | Yes |
| **Own fork, own stack** - not one shared cloud brain | Yes | At least a nod |
| **Non-commercial / community** | Yes | Yes |
| **How to get unstuck** - post error, no shame, async norms | Yes | Yes |
| **Personal goals** - human-chosen, AI-refined, shared through GitHub | Yes | Yes |

**OB1 (for copy - adapt wording, keep meaning):** *Open Brain* here means a personal, governed instance: code + memory habits you control, usually with git-first history, governed updates, and your own infrastructure. It is not a smarter ChatGPT tab; it is a repo-shaped workflow you run on your GitHub, with the [Cici](https://github.com/Xavier-x01/Cici) repo as the reference implementation and golden path.

**Purpose of cici-ai (for copy):** The group exists so hobbyists and curious builders can help each other through the first painful steps: fork, env, first session, no leaked secrets, first small win. It supports the Neural Sandbox / community intent: bring others along the same path Cici is walking, with no fee, no performance theater, and less drop-off by making questions normal.

**Personal goals (for copy):** Each person should choose one goal for their own OB1 instance to help solve or automate. The goal should come from real work, school, or personal life. Humans choose the goal, and AI can help refine it, make it more specific, and break it into smaller steps. Share progress through GitHub so the group can see visible work.

**Order of operations (recommended):** Post §1 -> pin -> then §2 with your client's notify-all option so early joiners and late joiners both see a clear purpose and a permanent anchor.

## 1) Pinned message - key group info

```text
🧠 cici-ai - who we are

This group is for people learning to run a real Open Brain (OB1) workflow: a governed, git-first companion setup on your GitHub, with your keys and your Supabase - not a shared "everyone in one account" toy.

Why we are here: help each other stand up the same stack without going alone (fork -> env -> first honest session), with Cici's public instance repo as the golden path to copy, not a black box.

Goals: each person chooses one personal goal for their own OB1 instance to help solve or automate. Humans choose the goal; AI can help refine it, make it more specific, and break it into smaller steps. The goal should come from real work, school, or personal life, and progress should be shared through GitHub.

⛏️ Golden-path repo: https://github.com/Xavier-x01/Cici
📘 Start here (repo overview): https://github.com/Xavier-x01/Cici/blob/main/README.md
🧭 Setup (Supabase + MCP, step-by-step): https://github.com/Xavier-x01/Cici/blob/main/docs/setup-guide.md
🌱 Instance / seed phase (governed init): https://github.com/Xavier-x01/Cici/blob/main/docs/seed-phase.md
📜 Governed state (how truth moves in Git): https://github.com/Xavier-x01/Cici/blob/main/docs/governed-state-doctrine.md
🔼 Upstream Open Brain (OB1 platform): https://github.com/NateBJones-Projects/OB1
🔒 Your setup: fork -> your GitHub, your own Supabase (no shared "everyone in one DB"), never post API keys or tokens in chat or in public files.
🧩 Tiers: Tier A = what's in Git as governed truth. Tier B = prepared context. Tier C = AI/DB recall - useful, not the same as approved business truth.
🧪 Sandbox: test on your fork; do not stress or experiment against the upstream repo as if it were your private playground for risky ops.
🤝 Norms: async, blame-free, ask in public if you can - we learn faster that way. Constructive > clever. Credit ideas.

We are not selling anything here. If you get stuck, post the error (screenshot ok). We would rather have a "dumb" question than silent drop-off. FAQ = README + setup guide + troubleshooting table in setup-guide - read those before DMing for basics.
```

## 2) Introductory message - purpose, next steps, gratitude, encouragement (@all)

```text
@all

Welcome to cici-ai - and thank you for being here.

What this is: we are a hobbyist room for Open Brain (OB1) - a governed, git-first way to run a serious companion-style stack on your repo and your infrastructure (not a generic AI chat). Cici is the public golden path we are learning on together (same patterns, your own fork): https://github.com/Xavier-x01/Cici

Purpose of this group: make it normal to be a beginner at this, normal to post errors, and faster to get unstuck than working alone. We are not selling a course; we are building in public where it is safe, and nudging each other through fork -> first session -> first small win.

Personal goals: each person should choose one goal for their own OB1 instance to help solve or automate. The goal should come from real work, school, or personal life. Humans choose the goal, and AI can help refine it, make it more specific, and break it into smaller steps. Share progress through GitHub so the group can see visible work.

Next steps (pick your pace, but do not let shame win):
1. Star + fork the golden-path repo: https://github.com/Xavier-x01/Cici
2. Follow setup on your GitHub (Supabase, env, never post secrets in chat) - the README is the map.
3. When you can, post your GitHub username and your personal goal, plus where you are (for example, fork done, or stuck on X). A short update beats a week of silence.
4. If you have evidence of progress, share a GitHub link, screenshot, commit, or post.

I am glad you showed up. The win is a small, honest step - we will cheer progress and debug errors together.
```

*Maraming salamat sa pagsama - small honest steps > perfect posts. If you are stuck, post the error, do not be shy.*
