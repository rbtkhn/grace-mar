# `cici-ai` group instruction block

Use this when you want to explain the work norm clearly to the group.

```text
Each person should choose one personal goal for their own OB1 instance to help solve or automate.

The goal must come from real work, school, or personal life. It should be something OB1 can realistically help with step by step.

Humans choose the goal. AI can help refine it, make it more specific, and break it into smaller steps.

The goal should produce visible work that the person shares with the group through their GitHub account.

Good goals are:
- simple
- specific
- measurable
- realistic for the next step

Each goal should answer:
- What problem are we trying to solve?
- How can OB1 help?
- What is the first small action?
- What will count as visible progress?

If the goal needs evidence, share a GitHub link, screenshot, commit, or post.

If the idea is too big, break it into a smaller goal.

If the idea is unclear, rewrite it until a new member can understand it quickly.

After the group agrees, post the goals and keep them updated in GitHub and chat.

Team practices:
- Use the shared timezone to keep check-ins and help requests timely.
- Keep updates short enough to read on a phone.
- Ask questions in the group instead of waiting too long in silence.
- Share screenshots, links, commits, or posts as proof of progress.
- Write instructions in simple English, with Filipino when it helps clarity.
- Keep the work tied to real school, work, freelance, family, or personal goals.
```
