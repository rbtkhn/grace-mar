## Mission (restated)

- **Self-teaching first:** Xavier uses **Cici** and repo habits to **learn without waiting** for frequent 1:1 guidance; the advisor weighs in **a few times a week**, not as the primary loop.
- **Train-the-trainer arc:** Build toward **teaching others** to stand up **their own** Claude + **Cici-style** workflow (fork, own Supabase, governed patterns).
- **Community, not fee:** Members join a **group** and work together; **no charge** for participation in this lane.
- **First numeric goal:** **Ten real people** in the group who **set up** their own Cici path (not “joined only”).
- **Context:** **Philippines** — everyday AI tooling still **not universal**; **early-mover** opportunity if onboarding is clear and repeatable.
- **Ethos:** **Bring others along** the same learning path Xavier is walking.

## Primary risk (operator choice): member drop-off

Drop-off is treated as the **main** failure mode (versus only advisor time or Xavier overload). It is driven less by missing README detail than by **shame**, **silent stalls** (Git, Supabase, first session), and **no felt “win”** before life interrupts.

### What usually kills the cohort

- **Fuzzy “done”** — “In the group” feels like progress; real milestone is **fork + own Supabase + no secrets leaked** (adjust labels to match pinned rules).
- **Silent failure** — Stuck users **don’t post** and vanish.
- **Long path to first dopamine** — If session one does not end with **something working** (even small), motivation dies.
- **Async shame** — Mitigated by **blame-free** norms and **modeling** questions in public (see Neural Sandbox rules).

### High-leverage mitigations (low meeting load)

1. **One shouted milestone** — Same definition in pinned message, checklist, and welcome: what **counts as set up**.
2. **Checkpoints** — e.g. Day 2 and Day 7: “Reply **done** or **stuck on X**” — **stuck** replies are the leads that matter.
3. **Smallest first win** — Order steps so **GitHub + fork** can ship **before** heavier MCP/Supabase if needed; or pair early steps with a **5-minute win**.
4. **Name silence** — “If you have not posted 48h after joining, you are probably stuck — **paste the error** (screenshot ok).”
5. **Structure beats scatter** — Ten solos drop more than **two waves of five** or **accountability pairs** with the same curriculum.

### Funnel metrics (honest)

Track **stages**, not only headcount:

| Stage | Example signal |
|-------|----------------|
| Joined group | In Telegram |
| GitHub account | Username posted (see transcript pattern) |
| Fork | Link to **their** fork |
| Supabase | Project created (self-report ok) |
| First working session | Define minimally (e.g. Claude Code opens repo, or MCP ping) |

Measure **where the stairs get steep**, not vanity joins.

Sparse guidance + high setup friction can still slide into **default helpdesk** unless the **group and docs** absorb repeat questions. **Train-the-trainer** only reduces load once **someone else** can run the golden path without Xavier. Treat **cohort one** as a **process test**, not only a recruitment tally.
