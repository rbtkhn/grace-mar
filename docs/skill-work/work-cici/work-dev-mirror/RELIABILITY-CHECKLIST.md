# Reliability checklist (xavier mirror)

Use this before shipping high-stakes drafts or process changes.

- [ ] Output action matches the reasoning (no "warn but approve" mismatch).
- [ ] Any factual claim has a traceable source file.
- [ ] Risky content has clear human approval/escation path.
- [ ] WORK content is not written into Record files.
- [ ] Candidate updates are staged, not merged, before approval.
- [ ] Final output states what is known vs uncertain.

If any box fails, mark **HOLD** and resolve before shipping.
