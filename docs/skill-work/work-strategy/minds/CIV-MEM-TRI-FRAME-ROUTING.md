## First opens (workflow)

1. Fix **entity X** (or question class that maps to a civ).
2. If present: `content/civilizations/<X>/MEM–RELEVANCE–<X>.md` — use as **dimension index**.
3. **MEM pulls (depth levels):**
   - **Minimum / shared spine:** open **1–2 MEMs** the index marks as primary for the question — gives everyone a common floor (read files; do not cite MEM IDs from memory).
   - **Tri-frame depth (recommended when running all three minds):** add up to **1–2 additional MEM reads per mind**, chosen so each mind’s **column** in the table above is actually fed — e.g. Barnes → MEMs touching **CONNECTIONS** / liability / enforcement; Mearsheimer → MEMs for **force, geography, incentives**; Mercouris → MEMs for **legitimacy, continuity, quoted evidence** (plus **`CIV–SCHOLAR–X`** / ARC **T** when narrative is load-bearing). Same MEM may serve more than one mind; you are **not** limited to one pair of files for the whole triad if the relevance map supports a split.
4. For current-decision framing: `content/civilizations/<X>/CIV–STATE–<X>.md` — especially sections on **material options**, **stability**, **decision-relevant history** (labels vary by template version).
5. **SCHOLAR** / **ARC** as needed for Mercouris (narrative, quotes) and Mearsheimer (structure).

**Tri-frame discipline:** keep a **shared core** (at least one overlapping passage or MEM so the three voices argue about the **same** case). Then let each mind lean on its **mind-specific** MEM pulls above — preserve tension; do not merge into one verdict.
