## 1. What this file is

This file is the human-readable source-governance layer for work-strategy.

It exists to help the operator decide which source types may be used for which strategy artifacts, how much trust to assign them, when stronger corroboration is needed, and when a source may support a historical or civilizational parallel.

This file does not replace the canonical weekly-brief registry that lives under work-politics. Instead, it governs how sources are used once material is being converted into work-strategy judgment, case-backed analysis, watch support, and decision framing.

## 2. What this file is not

This file is not:

- a duplicate of the work-politics weekly-brief registry
- a full bibliography
- a promise that high-tier sources are always correct
- a substitute for operator judgment
- a way to smuggle weak commentary into strong claims
- a Record or identity-facing surface

This file governs use, not truth.

## 4. Core rule

A source may be useful for framing without being sufficient for promotion.

The key question is not only "Is this source interesting?" but also:
- what artifact is being produced,
- what claim strength is being asserted,
- what corroboration is required,
- and what misuse risk must be controlled.

## 5. Source classes

Use the following source classes inside work-strategy.

### 5.1 Primary
Examples:
- official statements
- treaties
- laws
- agency releases
- military communiqués
- direct speeches
- public transcripts
- company filings
- central bank releases
- raw statistical publications

Typical value:
- best for direct claims about what an actor said, announced, filed, or formally did

Common weakness:
- self-serving framing
- omission
- propaganda
- incomplete context

### 5.2 Data
Examples:
- ACLED
- trade data
- customs data
- shipping data
- market data
- polling with disclosed methodology
- demographic and energy statistics

Typical value:
- best for trend validation, scale checks, and falsifying overconfident narratives

Common weakness:
- lag
- collection gaps
- measurement artifacts
- model dependence

### 5.3 Outlet / reporting
Examples:
- reputable newspapers
- defense reporting outlets
- beat reporting
- investigative outlets
- specialty foreign-affairs publications

Typical value:
- useful for event awareness, chronology, sourcing leads, and triangulation

Common weakness:
- editorial slant
- anonymous sourcing risk
- herd amplification
- narrative compression

### 5.4 Analyst / commentary
Examples:
- analysts
- newsletters
- expert interviews
- geopolitical commentary
- long-form interpretive essays

Typical value:
- useful for framing, mechanism generation, and hypothesis formation

Common weakness:
- interpretive overreach
- one-lens distortion
- motivated reasoning
- weak distinction between evidence and synthesis

### 5.5 Transcript / lecture / long-form discussion
Examples:
- podcast transcripts
- YouTube transcripts
- Predictive History
- long interviews
- roundtables

Typical value:
- useful for extracting structured hypotheses, recurring themes, civilizational frames, and question prompts

Common weakness:
- rhetorical momentum
- low evidentiary discipline
- speaker confidence mistaken for verification
- transcript noise

### 5.6 Operator note
Examples:
- strategy notebook entries
- resonance notes
- analogy audits
- hand-maintained watch notes
- session receipts

Typical value:
- useful for preserving accumulated judgment and local context

Common weakness:
- local bias
- drift
- hidden assumptions
- overfitting to recent concerns

## 6. Artifact-by-artifact usage policy

### 6.1 Neutral fact summary
Allowed:
- primary
- outlet/reporting
- data

Allowed with caution:
- transcript or analyst only when clearly labeled as interpretation or actor claim

Disallowed:
- presenting analyst or transcript rhetoric as neutral fact unless independently corroborated

Rule:
The neutral fact summary must separate:
- observed fact
- actor claim
- inference
- unresolved contradiction

### 6.2 Current-events analysis
Allowed:
- primary
- outlet/reporting
- data
- analyst
- transcript

Required discipline:
- case-index check before named historical parallel
- mismatch and falsifier if civilizational or historical layer is used
- analyst or transcript material must not silently substitute for empirical support

Rule:
Commentary may shape the question, but not automatically settle the answer.

### 6.3 Resonance note
Allowed:
- any source class, including transcript and operator note

Reason:
A resonance note is provisional and exploratory.

Required discipline:
- clearly label provisional status
- record mismatch
- record review trigger

Rule:
Resonance notes are where weak but interesting signals belong.

### 6.4 Analogy audit
Allowed:
- primary
- data
- outlet/reporting
- analyst
- transcript
- operator note

Required discipline:
- named mechanism
- strongest fit
- strongest mismatch
- falsifier
- use type: illustrative | framing-only | core model

Rule:
An analogy audit may use commentary heavily, but its output must still distinguish between historical illumination and historical overreach.

### 6.5 Watch support
Allowed:
- primary
- data
- outlet/reporting
- analyst
- transcript

Preferred support:
- at least one non-commentary support source for any watch that may shape prioritization or escalation

Rule:
A watch may begin from commentary, but meaningful watch support should ideally rest on corroborated signal rather than on one analyst's conviction.

### 6.6 Decision-point support
Allowed:
- primary
- data
- outlet/reporting
- analyst

Allowed with caution:
- transcript

Preferred standard:
- at least one primary or data anchor
- at least one corroborating reporting or second-source confirmation where practical

Rule:
A transcript or analyst may help frame options, but should not be the sole foundation for a material decision recommendation unless explicitly acknowledged as such.

### 6.7 Strategy-notebook entry
Allowed:
- all source classes

Required discipline:
- label whether the entry is fact-summary, interpretation, resonance, or strategy judgment
- cite named case or mechanism if using civilizational framing
- preserve uncertainty where the source base is thin

Rule:
Notebook entries are allowed to think ahead, but they should not pretend exploratory inference is verified fact.

### 6.8 Doctrine-note candidate
Preferred:
- multiple cases
- mixed source classes
- at least one stronger anchor set such as primary/data/reporting

Rule:
No doctrine note should rest primarily on one transcript, one analyst, or one hot-cycle news cluster.

## 7. Historical and civilizational use policy

Historical and civilizational framing is permitted in work-strategy, but source discipline tightens as claim strength rises.

### 7.1 Allowed uses
Historical or civilizational material may be used for:
- mechanism naming
- case-family selection
- fit/mismatch framing
- watch sharpening
- option framing
- falsifier design
- doctrine emergence testing

### 7.2 Disallowed uses
Historical or civilizational material should not be used to:
- declare inevitability
- bypass factual verification
- inflate weak analogy into strong model
- replace present-specific analysis with historical theater

### 7.3 Minimum rule
Whenever a historical or civilizational layer is used, record:
- case family or case ID
- mechanism
- why this fits
- why this may not fit
- falsifier
- confidence
- use type: illustrative | framing-only | core model

## 8. Promotion eligibility rules

A source may be interesting without being promotion-eligible.

Promotion-eligible use means the source can help support movement into:
- watch support
- decision-point support
- doctrine-note candidate
- optional gate candidate

### 8.1 Generally promotion-eligible
Usually eligible, depending on context:
- primary
- data
- strong reporting

### 8.2 Conditionally promotion-eligible
Eligible when corroborated or carefully bounded:
- analyst
- transcript
- operator note

### 8.3 Usually not sufficient alone
Usually not sufficient on their own for stronger promotion:
- single pundit commentary
- single long-form transcript
- isolated operator intuition
- anonymous rumor without independent support

Rule:
Promotion should track reuse, corroboration, and decision value, not eloquence.

## 9. Corroboration expectations by claim strength

### 9.1 Low-strength claim
Examples:
- "This may be beginning to rhyme with prestige-trap behavior."
- "This is worth a resonance note."

Needed:
- one plausible source or local analytical reason may be enough

### 9.2 Medium-strength claim
Examples:
- "This likely sharpens the watch."
- "This mechanism seems operative."

Needed:
- preferably at least two support points across classes, or one strong source plus one corroborating context source

### 9.3 High-strength claim
Examples:
- "This should alter recommendation."
- "This doctrine note is ready."
- "This is a core-model historical parallel."

Needed:
- mixed-source support where practical
- stronger anchors preferred
- mismatch and falsifier must be explicit

## 10. Use of transcripts and long-form discussion

Because work-strategy explicitly allows transcript ingest and Predictive History-style inputs, transcript discipline must be unusually clear.

### 10.1 What transcripts are good for
- hypothesis generation
- theme extraction
- civilizational framing
- identifying candidate mechanisms
- generating follow-up questions
- spotting recurring narrative structures

### 10.2 What transcripts are bad for
- neutral fact authority
- precise chronology without verification
- decisive support for strong recommendations by themselves

### 10.3 Transcript rule
Treat transcript material as:
- interpretation,
- argument,
- or actor claim,

unless it is independently verified.

### 10.4 Promotion rule for transcript-derived material
Transcript-derived material may support:
- resonance notes
- analogy audits
- notebook entries
- watch support

It should not be the sole support for stronger decision or doctrine claims unless explicitly marked as provisional and exploratory.

## 11. Weak-signal policy

Weak signals are allowed in work-strategy because early pattern detection is part of the lane's purpose.

But weak signals must remain clearly bounded.

When a weak signal enters:
- mark it as weak
- record what would upgrade it
- record what would kill it
- avoid smuggling it into synthesis as established trend

Weak signals are excellent raw material for:
- resonance notes
- case-hit candidates
- analogy audits

They are poor standalone foundations for:
- hard recommendations
- doctrine claims
- strong historical parallels

## 12. Source-tier interpretation

Use [source-tiers.md](source-tiers.md) for numeric or labeled trust-tier meanings.

Working rule:
- source tier informs starting confidence
- artifact type determines required discipline
- corroboration depends on claim strength, not only source prestige

Do not confuse source prestige with automatic permission to overclaim.

## 14. Review protocol

When reviewing source use in work-strategy:

1. Ask what artifact is being produced.
2. Ask what claim strength is being asserted.
3. Ask whether the source class is appropriate to that artifact.
4. Ask whether corroboration is needed.
5. Ask whether civilizational or historical framing is being overclaimed.
6. Ask whether the source improved judgment or only decorated it.
7. Downgrade the artifact if the source base is too weak.

## 15. Minimal maintenance standard

This file is healthy when:
- it clarifies what source types are for
- it reduces source misuse in current-events analysis
- it keeps transcript material useful without letting it dominate fact claims
- it supports historical and civilizational work without turning that work into theater
- it stays lighter than a full canonical bibliography regime

This file is unhealthy when:
- everything becomes promotion-eligible
- transcript rhetoric keeps leaking into neutral summaries
- historical framing outruns source discipline
- source prestige is treated as a substitute for reasoning
- the file duplicates work-politics instead of complementing it

## 16. Operator prompt

When unsure how to use a source, ask:

What artifact am I making, what claim strength am I asserting, and is this source class actually strong enough for that move?
