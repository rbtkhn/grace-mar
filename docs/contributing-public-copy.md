# Contributing — public copy tone and examples

**Audience:** Anyone editing **README**, marketing blurbs, **issue templates**, or **skills** that face **maintainers or the public** outside this repo.

**Companion Record rules** still live in [AGENTS.md](../AGENTS.md). This doc is **tone and safety for outbound prose**.

---

## Compare on attributes, not labels

When positioning Grace-Mar (or reviewing copy that mentions other projects):

- Prefer **deployment model, governance, threat assumptions, and merge authority** over **character judgments** about other tools or teams.
- Keep disagreement **factual** and **scoped** — readers may use those projects productively.
- Avoid **pejorative** or **viral-culture** framing unless there is a documented safety reason (then cite the fact, not the vibe).

---

## Example secrets and tokens in docs

- Use **obviously fake** placeholders (`sk-example-replace-me`, `xxxxxxxx`, `REDIRECT_URI_EXAMPLE`).
- Add **one line** where it helps: values are **illustrative**; rotate anything that was ever real.
- Do not paste **live** API keys, bot tokens, or companion-only paths into **public** docs or **portable** skill cores meant for export.
