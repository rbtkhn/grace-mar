# THINK — capability taxonomy

**Purpose:** Structured **capability types** for THINK claims and observation — not a curriculum. Use with [think-levels.md](think-levels.md).

**Territory:** SKILLS / THINK only; identity promotion still via gate.

## Types

| `capability_type` | Meaning |
|-------------------|---------|
| `comprehension` | Can follow and restate material at appropriate depth |
| `recall` | Can retrieve facts or patterns from prior exposure |
| `comparison` | Can contrast two sources, periods, or frames |
| `abstraction` | Can state patterns or principles beyond single examples |
| `analogy` | Can map structure across domains (use with analogy-audit discipline in work-strategy when public) |
| `question_generation` | Can ask productive clarifying or extension questions |
| `causal_reasoning` | Can trace plausible cause → effect with stated uncertainty |
| `uncertainty_handling` | Can hold “don’t know yet” / confidence bounds appropriately |
| `topic_familiarity` | Recognizes vocabulary and references in a domain |
| `vocabulary_acquisition` | Active lexicon growth (languages or domain jargon) |
| `transfer_across_domains` | Applies a move learned in one context to another |

Validator enums in [think_claims.schema.json](../../schemas/skill_think/think_claims.schema.json) should stay aligned with this table.
