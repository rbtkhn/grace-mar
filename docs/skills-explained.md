# Skills explained — two layers

**Purpose:** Disambiguate **portable operator skills** from **SKILLS** (Record capability). OB1-flavored users often expect “skills” = executable packs; Grace-Mar uses both ideas.

> Grace-Mar is not just a memory layer; it is a **governed companion record**. **Library, Skills, Evidence** name Record surfaces; **portable skills** name reusable operator assets.

---

## Portable skills (execution / operator)

- **Location:** [skills-portable/README.md](../skills-portable/README.md), `.cursor/skills/`, discovery ladder in portable README.
- **Role:** Reusable **workflows** for Cursor/agents — drafts, staging, scripts — **not** automatic Record truth.
- **Governance:** Does not replace the gate; may **stage** candidates only.

---

## SKILLS (Record — what the fork evidences it can do)

- **Location:** `users/grace-mar/self-skills.md` (canonical capability index; legacy `skills.md` until migrated).
- **Role:** **THINK** and **WRITE** (and related) **capability containers** — what the Record can show the companion can do, with evidence discipline ([skills-modularity.md](skills-modularity.md), [skill-think/README.md](skill-think/README.md)).
- **Governance:** Capability claims that matter for identity/product **enter through the Approval Inbox** when updating governed content — same merge rule as SELF/EVIDENCE ([AGENTS.md](../AGENTS.md)).

---

| | Portable skills | SKILLS (Record) |
