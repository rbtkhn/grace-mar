# System Tensions and Mysteries

This is the canonical living document of the **hard challenges and open mysteries** that the companion-self system must carry.  
It is **not** a bug list. It is the operating manual for the system’s own recursive self-awareness.

Every instance inherits this file and is encouraged to add local annotations while preserving the core list.

## Ten Hard Challenges

1. **Boundary precision**  
   Distinguishing “operational memory” from “Record truth” sounds simple, but in live workflows signals blur quickly.

2. **Layer drift over time**  
   `self-*`, `skill-*`, `work-*`, and `docs/` can slowly overlap unless continuously policed.

3. **Gate discipline under speed pressure**  
   The faster workflows get, the easier it is to accidentally bypass the merge membrane conceptually.

4. **Identity vs intent tension**  
   Core purpose is durable, but execution intent changes fast; keeping both coherent is a constant balancing act.

5. **Human-readable vs machine-auditable state**  
   Docs can say the right thing while scripts/provenance lag (or vice versa).

6. **Template/instance divergence**  
   Sovereignty is a feature, but it makes “alignment without overwrite” technically and socially hard.

7. **Confidence calibration in bridges**  
   Turning messy markdown into reliable planning signals without overfitting is tricky.

8. **Voice authenticity without leakage**  
   Keeping style vivid while never importing undocumented knowledge is a subtle, ongoing constraint.

9. **Schema evolution without breakage**  
   Every useful new surface risks compatibility churn across repos.

10. **Operational truth density**  
    There is so much context that deciding what must persist vs what should decay is still more art than science.

## Greatest Mysteries (still open)

- What is the **minimal memory** that preserves continuity without ossifying the system?
- How much **structure** helps growth before structure itself becomes friction?
- Can we formalize “**maturity**” enough to auto-tune guidance without violating sovereignty?
- Where exactly is the boundary between a **living fork** and a toolchain around a person?
- What is the right **long-term unit of progress**: evidence volume, behavioral change, or identity coherence?

## Evolution Rules

- Preserve the original 10 + 5 verbatim in every major version.
- Instances may add local annotations (e.g., "Grace-Mar status: ...").
- Template upgrades may only add new sections or refine rules — never delete or soften the core list.
- Any new mechanism must be stress-tested against all 10 challenges.

**Status:** Living. Not solved. Actively carried.
