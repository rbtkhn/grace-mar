# Persian Regime-Adaptive Strategy

**Preferred activation (operator):** say **`regime strategy`**. **Aliases:** **`persian strategy`**, **`regime switch`**.

**Purpose:** Apply three operating modes derived from Persian civilizational continuity (Achaemenid → Parthian → Sassanid) to current strategy work. Each mode is a lens for evaluating posture, not a prescription. The operator selects the mode; the agent applies its logic to the material at hand.

**Source corpus:** `CIV–CORE–PERSIA` / `state-persia` via SELF-LIBRARY / CIV-MEM. This is WORK territory content — not Record, not Voice, not companion identity.

## Three operating modes

| Mode | Historical anchor | Core logic | When to use |
|------|-------------------|------------|-------------|
| **Tolerance** | Achaemenid (Cyrus) | Integration through recognition; local legitimacy retained; many peoples at one gate; face over category. | Opening posture; coalition-building; multi-actor coordination; when diversity is the asset. |
| **Parity** | Parthian / Roman frontier | Sustained engagement without absorption; mutual deterrence; decentralized resilience; "neither can permanently destroy the other." | Protracted competition; peer rivalries; when neither side can dominate; endurance over victory. |
| **Compression** | Sassanid / post-conquest | Defeat densifies into doctrine; form reconstitutes; sovereignty as sacred; siege literacy; survival as primary metric. | Defensive consolidation; resource scarcity; institutional hardening; when the environment is hostile and the priority is continuity. |

## When to use

- The operator is working a **strategy pass** and wants to stress-test posture against civilizational patterns.
- A current situation maps to one of the three modes and the operator wants to **name** which mode is active and why.
- Comparing **regime transitions** — when should posture shift from tolerance to parity, or from parity to compression? The historical record provides transition triggers.

## Agent behavior

1. **Read the current material** (daily brief, notebook entry, framework, or operator prompt).
2. **Identify which mode best fits** the current situation. Present a one-line rationale for each mode's relevance (or irrelevance).
3. **Wait for operator selection.** Do not develop the full analysis until the operator picks a mode or combination.
4. **Apply the selected mode:** Reread the material through that lens. Produce a bounded analysis (500–800 words) covering:
   - What the mode's logic predicts about the current situation
   - Where the historical pattern maps cleanly and where it breaks
   - Transition triggers: what signals would indicate a shift to a different mode
5. **Link back** to CIV-MEM source entries when a specific MEM file grounds the claim.

## Transition triggers (historical)

These are the patterns that historically forced mode shifts:

| From → To | Trigger |
|-----------|---------|
| Tolerance → Parity | External peer emerges that cannot be integrated (Rome); recognition fails; arms race begins. |
| Tolerance → Compression | Catastrophic defeat (Alexander); elite defection; system optimized for stability breaks under tempo. |
| Parity → Compression | Exhaustion without resolution; internal fracture; resource depletion from sustained frontier. |
| Compression → Tolerance | Survival secured; new dynasty reconstitutes legitimacy; capacity for integration returns (Safavid revival). |
| Compression → Parity | Partial recovery; enough strength to deter but not to integrate; equilibrium at reduced capacity. |

## Anti-patterns

- **Treating a mode as permanent identity** — the value is in switching, not in locking a label.
- **Forcing a mode onto material that doesn't fit** — if none of the three maps cleanly, say so; not every situation is Persian.
- **Using this as Record content** — this is an operator strategy tool, not companion knowledge or identity.
- **Pre-developing all three modes before selection** — present labels, wait for the pick.
