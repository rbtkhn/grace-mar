# Daily brief regen + post-process merge

**Preferred activation (operator):** **`daily brief regen`**. **Aliases:** **`merge daily brief tail`**, **`brief-date regen`**, **`regen brief keep 2c`**.

**Purpose:** Re-run `generate_work_politics_daily_brief.py` for a historical **`daily-brief-YYYY-MM-DD.md`** filename without losing operator-authored narrative retrofit material that lives **after** the generated spine — anchored at **`## 2c. Narrative layer retrofit`**.

**Not** Record truth. **Not** a substitute for filling **`## 1d`–`## 1h`** actor-primary passes.

## Preconditions

1. Target brief exists: `docs/skill-work/work-strategy/daily-brief-YYYY-MM-DD.md`.
2. The brief already contains a post-process tail beginning with **`## 2c. Narrative layer retrofit`** (through end-of-file). If missing, run a narrative pass first per `docs/skill-work/work-strategy/daily-brief-template.md`.

## Procedure

### 1) Choose fetch mode

- **Offline / stable structure regen:** pass **`--no-fetch`** to skip RSS (recommended when you only want template/config snapshot changes merged).
- **Live refresh:** omit **`--no-fetch`** — expect headline drift vs the original day’s capture.

### 2) Merge regen + tail (recommended)

```bash
python3 scripts/merge_daily_brief_postprocess.py -u grace-mar --date YYYY-MM-DD --no-fetch
```

- Default target path: `docs/skill-work/work-strategy/daily-brief-YYYY-MM-DD.md`.
- If the regenerated base already contains a **`## 2c. Narrative layer retrofit`** block, the script **refuses** to duplicate unless you pass **`--force`** (it strips an existing `## 2c` tail from the fresh generator output before splice).
- The script **sanitizes** the preserved tail: it drops a stray duplicate **`## 3`–`## 7`** stub block when it sits immediately before **`## 8. Quality rubric footer`**, and removes a **trailing** `_Generated by … daily-brief-config.json` line if the tail still carried a second copy of the generator signature (the fresh base already ends with one before **`## 2c`**).

### 3) Manual regen (advanced)

```bash
python3 scripts/generate_work_politics_daily_brief.py -u grace-mar \
  --brief-date YYYY-MM-DD --no-fetch \
  -o /tmp/daily-brief-YYYY-MM-DD.regen.md
```

Then splice tail manually (error-prone — prefer the merge helper).

### 4) Optional — append inbox handoff bundle

If the brief’s **`### Handoff to Grace-Mar`** block contains paste-ready lines, append them under the accumulator append line in:

`docs/skill-work/work-strategy/strategy-notebook/daily-strategy-inbox.md`

Use an HTML comment guard to keep idempotency:

`<!-- brief-handoff-bundle: YYYY-MM-DD — from daily-brief-YYYY-MM-DD.md ## 2c -->`

## Evidence / verify discipline

- Prefer **`verify:wire-RSS`** on RSS-sourced inbox lines.
- Use **`verify:OSINT-unverified`** for social / chain captures.
- Use **`verify:tier-A`** / **`verify:operator-transcript`** only when those notebook conventions apply.
- Any hard number or attributed quote without a primary still requires **`[UNVERIFIED — no primary found]`** in narrative prose (see `docs/skill-work/work-strategy/grok-system-preamble.md`).

## Boundaries

- **WORK only** — strategy lane artifacts; not SELF, not EVIDENCE, not Voice.
- Do not treat regenerated RSS as “more true” than prior captures without an explicit compare note.
