# Interface artifacts (derived)

This directory stores generated or staged **interface artifacts** and any lightweight metadata files that describe them.

Policy:

- artifacts here are **derived WORK outputs**
- they may be deleted and regenerated
- they have **no Record authority**
- screenshots, logs, and metadata here do **not** prove external claims
- use Workbench when artifact behavior matters and you want an inspection receipt

Storage note:

- use this directory for **new cross-lane prototypes**
- keep artifacts in an existing lane-specific bucket when that lane already has one, such as `artifacts/work-strategy/strategy-notebook/`
